# 2026-06-07 — Imgy Bot: много правок

## Стили + аватар
- **Стили доступны только если выбран аватар.** Если «Без аватара» — показывает сообщение «выбери аватар» с кнопкой.
- Режим «Без аватара» сохраняется после `/start` (не сбрасывается на первый аватар).

## Промпт
- Убрано дублирование сообщения о генерации (было 2: «✍️ Генерирую: ...» + «✍️ Генерирую по твоему описанию...»).
- Починен **Повторить** — убрано двойное списание генераций (consumed in prompt_repeat handler + again in generateCustomAvatarWithPhoto).
- При нажатии «Повторить» отправляется явное сообщение в чат.
- Добавлены описания кнопкам после генерации (Повторить / Выйти).

## Модели и ошибки
- **Flash 2.5** — блокировка совместимости с «Без аватара» сначала добавлена, потом убрана по просьбе.
- Улучшен промпт для text-to-image: `<start_of_image_generation>` теги.
- Добавлено логирование полного candidate от Gemini при ошибке.
- Обработка всех finishReason Gemini:
  - `NO_IMAGE` → «Не смогли сгенерировать изображение, попробуйте еще раз.»
  - Остальные → «Не смогли сгенерировать из-за ограничений нейросети. Попробуйте переформулировать запрос или использовать нейросеть Про.» (для Про модели — без упоминания Про).
- Функция `finishReasonMessage` — принимает `settings` для проверки модели.
- Убран дублирующийся префикс «❌ Не удалось сгенерировать: » из сообщений.

## Удалён мёртвый код
- Удалён `generateNoAvatar` (стили без аватара — недоступны, метод не нужен) и его вызов в bot-runner.js.
