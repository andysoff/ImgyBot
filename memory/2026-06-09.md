# 2026-06-09

## Avatar Bot — Подключение оплаты через ЮKassa

### Что сделано
- Подключён модуль `payments.js` для создания реальных платежей через ЮKassa (ранее был демо-режим)
- Добавлен `handleBuyMenu` — при открытии меню покупки сразу создаются платежи для всех пакетов, кладутся ссылки в кнопки
- Добавлен `startPaymentWatcher` — фоновая проверка статуса платежа каждые 8 сек, до 225 попыток (~30 мин)
- При старте бота watcher'ы восстанавливаются для незавершённых платежей
- Демо-режим сохранён для случая, когда нет ключей ЮKassa (автоначисление через 2 сек)
- Убрана кнопка "✅ Я оплатил" — теперь только авто-проверка
- Убран текст про "Оплата не подключена"

### Рефакторинг хранения
- Платежи вынесены из `conversations.json` в отдельный файл `data/payments.json`
- Функции: `getPendingPayments`, `addPendingPayment`, `removePendingPayment`, `getAllPendingPayments`
- При рестарте watcher'ы восстанавливаются из `payments.json`
- `conversations.json` теперь чистый — только состояние диалога

### Обнаруженные проблемы
- Платеж `31ba7af4-000f-5001-9000-18efb0062097` (100₽, 10 ген) был оплачен, но не начислен — watcher не сохранился между рестартами. Зачислено вручную.
- Итого в ЮKassa: 5 успешных оплат по 100₽ = 500₽, 50 генераций
- 23 pending, 8 отменённых
