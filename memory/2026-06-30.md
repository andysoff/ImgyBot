# 2026-06-30 — Imgy Avatar Bot

## OpenAI gpt-image-2 — фото-референс через edits endpoint

**Проблема:** gpt-image-2 с фото-референсом слал `images: [file_id]` в `/v1/images/generations` — API возвращал 400 `Unknown parameter: 'images'`. Параметры `images`/`image` в generations endpoint не поддерживаются ни для какой модели.

**Решение:** Все модели с file_id теперь идут через `/v1/images/edits`, где `images: [{ file_id: "..." }]` — корректный формат. Убран isV2-бифуркация endpoint'ов.

**Файлы:** `scripts/generate-image-openai.js` — функция `generateFromPhotoWithFileIds`.

## Файловый кеш

- Первая загрузка фото: `uploadToAll()` → грузит в Gemini File API (resumable upload) + OpenAI File API (multipart) параллельно
- Результаты (gemini.uri + openai.fileId) сохраняются в `avatars.json` → `sourceFiles[]`
- При повторных генерациях: только проверка статуса (GET к API провайдера), без перезагрузки
- Если файл протух — перезагружается только для текущей модели (не оба)
- Код: `ensureSourceFiles()` в `bot-runner.js`
