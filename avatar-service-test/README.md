# Imgy — Avatar Service

Сервис генерации аватарок. Бот в Telegram.

## Структура

```
avatar-service/
├── data/
│   ├── users.json          — пользователи + счётчики генераций
│   ├── avatars.json        — аватары (у юзера может быть >1)
│   ├── styles.json         — стили генерации
│   └── conversations.json  — состояния диалогов с клиентами
├── photos/                 — папки userId/avatarId/photo_N.jpg
├── scripts/
│   ├── user-manager.js     — управление пользователями (админ-скрипт)
│   └── bot-logic.js        — логика диалогов с клиентами
└── README.md
```

## Счётчик генераций

- **На уровне пользователя** — общий на все аватары
- **Старт:** 10 (разовый бонус при первом создании)
- **Расход:** −1 при каждой успешной генерации
- **Блокировка:** при 0 генерации не выполняются
- **Пополнение:** через администратора (команды чата)
- **Управление через чат (админ):** `пополни user_1 20` / `установи user_2 5`

## Сценарий клиента (новый flow)

1. Клиент пишет боту → **приветствие**: "Привет! Я Imgy, … 10 бесплатных генераций. Загрузи свои фото."
2. Клиент отправляет **фото одним сообщением** (до 10 шт.)
3. Фото сохраняются → создаётся пользователь (если новый) → счётчик = 10 → **кнопки со стилями**
4. Клиент выбирает стиль → генерация → −1 генерация
5. При 0 → "Генерации закончились. Напиши администратору."

## Диалоговые состояния

| Состояние | Что происходит |
|---|---|
| `idle` | Ожидание, нет активного диалога |
| `awaiting_photos` | Бот ждёт фото (одно сообщение, до 10 шт.) |
| `awaiting_style` | Бот показал стили (inline-кнопки), ждёт выбор |

## Стили

| id | Название | Описание |
|---|---|---|
| `portrait` | Портрет | Классический портрет, акцент на лице |
| `sport` | Спорт | Спортивный образ, динамика |
| `in_car` | В машине | За рулём или в салоне автомобиля |
| `at_work` | На работе | Рабочая обстановка, офис или кабинет |

## Админ-скрипт

```bash
node scripts/user-manager.js list                          # список юзеров
node scripts/user-manager.js get user_1                    # инфо о юзере
node scripts/user-manager.js add-generations user_1 20     # пополнить
node scripts/user-manager.js set-generations user_1 5      # установить
```
